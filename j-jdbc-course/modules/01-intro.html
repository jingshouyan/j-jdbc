j-jdbc 是什么？
让数据库操作变简单

你是否曾经用 Java 写过数据库代码？如果你写过，你一定经历过这个循环：打开连接 → 创建语句 → 执行查询 → 处理结果集 → 关闭连接 → 关闭语句 → 捕获异常……重复几十次。更糟糕的是，你的代码里到处都是 SQL 字符串，拼错一个字就会在运行时崩溃。

j-jdbc一个基于 Spring JDBC Template 的 CRUD 增强框架，通过注解自动生成 SQL 语句，大幅减少数据库操作的样板代码。 就是为了解决这个问题而生的。它是一个轻量级的 Java 库，工作方式是："你告诉我数据结构，我来搞定数据库。"

「声明式」的魔法

j-jdbc 的核心思想是 声明式编程：你不用告诉代码"怎么"做（写 SQL），你只需要声明"是什么"（用注解描述数据）。框架会自动生成所有 SQL。

想象一下：你创建一个普通的 Java 类，加上几个注解，然后——嘭！——完整的增删改查、分页查询、条件过滤全部自动可用。不需要写一行 SQL。

🧩 隐喻：智能家居

传统 JDBC 编程就像手动操作家里每一个电灯开关——你得知道每个开关在哪、怎么按、哪个控制哪个房间。j-jdbc 就像一个智能家居系统：你只需要告诉它"客厅需要灯光"（声明一个带注解的对象），它就会自动帮你打开正确的开关、调节亮度、甚至在日落时自动亮起（自动生成 SQL、管理连接、处理分页）。

一行代码，一个世界

这是 j-jdbc 最令人惊叹的地方：你只需要 4 个文件 就能完成一个完整的数据库操作。

java // 1️⃣ 数据库对象 — 加注解描述表结构 @Table(value = "DEMO_USER", comment = "用户表") public class UserDO extends BaseDO { @Key @Column(comment = "主键") private String id; @Column private Integer age; @Column(value = "NICK_NAME_TT4") private String nickname; } // 2️⃣ DAO 接口 — 就一行！ public interface UserDao extends BaseDao<UserDO> {} // 3️⃣ DAO 实现 — 也是一行！ @Repository public class UserDaoImpl extends BaseDaoImpl<UserDO> implements UserDao {} // 4️⃣ 使用 — 直接调用 userDao.insert(user); // 插入 userDao.update(user); // 更新 userDao.find("U123@abc"); // 按主键查询 userDao.query(conditions); // 条件查询 userDao.delete("U123@abc"); // 删除

看到没？没有 SQL，没有连接管理，没有资源释放。 所有的数据库细节都被封装在 `BaseDaoImpl` 中。你可以把精力放在业务逻辑上，而不是重复编写 CRUD 代码。

📦 j-jdbc-common — 共享工具箱

注解定义（@Table, @Column, @Key, @Index）、实体基类（BaseDO）、条件查询构建器（ConditionUtil）、数据模型（TableInfo, ColumnInfo）—— 所有模块共享的基础。

⚙️ j-jdbc-core — 引擎核心

BaseDao/BaseDaoImpl、SQL 生成器（支持 MySQL）、AES 加密、雪花 ID 生成器、DML 事件总线。这是整个框架的大脑。

🚀 j-jdbc-starter — Spring Boot 自动配置

自动建表（3 种模式）、数据库 ID 生成器、SQL 日志 AOP —— 开箱即用的 Spring Boot Starter。

🤯 啊哈！时刻

j-jdbc 的命名本身就是一种聪明： "j" 既是 "Java" 也是你名字的缩写；"jdbc" 精确地指向了它的根基 —— Spring JdbcTemplate。没有选择 Hibernate 那样的重量级 ORM，也没有用 MyBatis 那样的 XML 映射，而是恰到好处地在 JdbcTemplate 之上增加了一层注解驱动的便利。
