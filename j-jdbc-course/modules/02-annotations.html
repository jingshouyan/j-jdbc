用注解说清数据
对象到数据库的映射

如果说 j-jdbc 是一门语言，那么 注解（Annotation） 就是它的词汇。你不需要学习 SQL 的语法，也不需要配置 XML 映射文件——只需要在 Java 类上贴标签，框架就知道你想怎么存储数据。

想象一下：你有一个"用户"的概念，它有名字、年龄、标签。在数据库中，它是一个表；在 Java 中，它是一个类。j-jdbc 的注解就是连接这两个世界的桥梁。

@Table — 给类穿上数据库的外衣

@Table表注解。加在类上，告诉 j-jdbc 这个 Java 类对应数据库中的哪张表。如果不指定表名，默认使用类名。 是最顶层的注解。它告诉框架："嘿，这个 Java 类对应一张数据库表。"

java @Table( value = "DEMO_USER", // 数据库中实际的表名 comment = "用户表" // 建表时用的备注 ) public class UserDO extends BaseDO { // 字段... }
☕ Java 代码 中 文 解 释
@Table(value = "DEMO_USER")
"这个类对应数据库中的 DEMO_USER 表"
@Table(comment = "用户表")
"给这个表加个备注：用户表"
public class UserDO
UserDO 就是这张表的 Java 化身
🤯 啊哈！时刻

注意看 "Table" 注解的拼写： 它没有写成标准的 "Table"，而是使用了正确的拼写 "Table"。开发者通过这种一致性设计告诉我们：注解的命名可以直接映射到其作用——要让名称清晰到不需要文档也能猜到用法。

@Column — 精确控制每一列

@Column列注解。加在字段上，定义数据库列名、长度、类型、是否加密、是否 JSON 存储等属性。如果不指定，默认使用字段名作为列名。 是 j-jdbc 最强大的注解。它不仅能控制列名和长度，还能指定加密方式、JSON 序列化、数据类型推断等高级功能。

java // 简单用法 — 列名等于字段名 @Column private Integer age; // 指定列名和备注 @Column(value = "NICK_NAME_TT4", comment = "昵称") private String nickname; // JSON 格式存储 + 加密 @Column(json = true, encryptType = EncryptType.FIXED, encryptKey = "abcd", length = 1000) private List<String> tags; // 不可变字段（创建后不能改） @Column(immutable = true, order = 1001, comment = "创建时间") private Long createdAt;

数据类型自动推断： j-jdbc 会查看字段的 Java 类型，自动选择最合适的 MySQL 数据类型。`int` → INT, `String` 且长度小于 5000 → VARCHAR, `String` 且长度 ≥ 5000 → TEXT, `BigDecimal` → DECIMAL, `LocalDateTime` → DATETIME, `List` 且 `json=true` → JSON……你不用记住这些映射，框架会帮你处理。

@Key — 主键 & @Index — 索引

java // 主键 — 只能是 Long 或 String @Key(generatorIfNotSet = true) // 如果没设置值，自动生成 @Column(comment = "主键") private String id; // 单字段索引 @Index private Integer age; // 复合索引（类级别） @Index({"age", "nickname"}) @Index({"id", "name"}) public class UserDO ...

主键（@Key） 是每个表的灵魂。j-jdbc 要求主键只能是 Long 或 String 类型。如果设置了 `generatorIfNotSet = true`，当你插入数据时没有设置主键值，框架会自动生成一个唯一 ID。

索引（@Index） 可以在字段上（单字段索引）或类上（复合索引）使用。数据库索引就像是书的目录——它让查询速度从"逐页翻找"变成"直接翻到正确的页码"。在第 4 课中，我们会详细讲解主键是如何生成的。

注解们的群聊时间 💬

想象一下，当 j-jdbc 启动时，注解们正在群里讨论今天的工作：

T
@Table 大家早上好！我是 DEMO_USER 表的注解。今天我们的任务是搞定新来的 UserDO 类。
C
@Column 收到！我已经把 age 映射成了 INT，nickname 是 VARCHAR(255)，tags 是 JSON 类型。
K
@Key 主键 id 归我管！String 类型，如果用户没提供，我就让 ID 生成器自动分配一个。
I
@Index 好的，我已经给 age 创建了索引。再加上 (age, nickname) 的复合索引——这样按年龄和昵称查就飞快了！
T
完美！@TableInfo 你把我们的配置整理成建表 SQL，发给 SqlGenerator4Mysql 执行吧。
S
SqlGenerator 收到！CREATE TABLE IF NOT EXISTS DEMO_USER (id VARCHAR(50), age INT, nickname VARCHAR(255), tags JSON, ...) COMMENT='用户表' —— 搞定！

幕后引擎：TableInfo 是如何工作的

所有的注解魔法都归结于一个核心类——TableInfo。它读取类上的所有注解、字段、以及父类的字段，构建出一张完整的"元数据地图"。

☕ TableInfo 构造函数 它做了三件事
tableName = clazz.getSimpleName();
1️⃣ 先以类名作为默认表名
@Table table = clazz.getAnnotation(Table.class);
2️⃣ 检查是否有 @Table 注解 → 有的话覆盖表名
Field[] fields = c.getDeclaredFields();
for (Field field : fields) {
ColumnInfo beanColumn = new ColumnInfo(field);
addColumnInfo(beanColumn);
}
3️⃣ 遍历所有字段→每个字段变成 ColumnInfo 对象→添加到地图中

注意：它会遍历整个类继承树（从当前类一直往上到 Object），确保父类如 BaseDO 中的 createdAt、updatedAt、deletedAt 字段也被纳入。
